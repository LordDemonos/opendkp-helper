Bidding

Automatic API bidding on active auctions. Available in both Raid Leader and Raider modes — keep an opendkp.com tab open while enabled.

Shows past Bid Results wins (Include All) for active auctions, with an estimated value and a small price chart. Use Search… in the dropdown to look up any item. Requires API sign-in.

Sign in with your OpenDKP account (same as the website). Credentials are stored locally and shared with Raid Leader API tools if you switch modes.

Uses guild subdomain and API sign-in from OpenDKP Raids in the Raid Leader section below.

While an opendkp.com tab is open, polls active auctions and bids in increments until you are in a winning slot (top bid for single items, top N for quantity N), your max DKP for that item is reached, or you run out of DKP. If a full increment would overshoot your max, it bids the remaining amount up to that max (e.g. leader at 380, increment 50, max 400 → bids 400). When a matching auction has 30 seconds or less remaining, polling speeds up to every 2 seconds. Character rank comes from Refresh characters. Rank max bids come from Bidding Tool → Bid Rules (select each rank once; values merge into a local cache and are included in Backup & Restore). When your character is called as the winner for a matching item, that rule is automatically disabled (re-enable the checkbox to bid on the same item again).
Only alerts when you win an auction (detects your characters from the header). Automatically enabled for Raider profile.

Alerts & Sounds

Notification sound

Choose your preferred notification sound. Custom sounds require uploading an audio file.

Custom sounds

Upload MP3, WAV, or OGG files. Maximum 3 custom sounds. Suggested size ≤ 100 KB each for quick notifications. Files are stored locally in your browser.

Text-to-Speech

1.0x
Available variables:
{winner} - Winner name
{bidAmount} - Bid amount
{itemName} - Item name
{winners} - Comma-separated winners list
{isRollOff} - true/false
{multipleWinners} - true/false
Text-to-speech announces auction completions after the sound plays. Requires browser permission.

Read New Auctions

On days:
Speaks "New auction: Item Name" when a new auction appears. Uses your TTS voice and speed settings. Respects Quiet Hours.

Item Watchlist Alarm

When a watched item is posted for bid: plays alarm.mp3 for 5 seconds, flashes the OpenDKP page red once per second (5 times), and speaks the item name aloud every second. Works for both Raider and Raid Leader modes. Ignores Quiet Hours. Test Alarm runs the full effect on an open opendkp.com tab; otherwise plays audio/TTS here only.

Quiet hours

Quiet Hours disables sound notifications during the selected time range.

Visual & detection

Visual alerts help ensure you don't miss timer completions. Use the Sound Volume / TTS Test buttons above to check audio. Winner and roll-off alerts are verified on a live opendkp.com auction (ties at rank caps like 400 or 1000 are detected on the page, not from Settings).
Raid Leader

OpenDKP Raids

Raid Leader mode only. Connect to the OpenDKP HTTP API using your guild subdomain (from https://YOURGUILD.opendkp.com → enter YOURGUILD).

Account

Lowercase letters, numbers, hyphens only. Never shared between users—stored only in your extension settings.
Saved in extension local storage only (not synced). Used for API sign-in.

Not the website login. This calls AWS Cognito (same flow as the Postman collection) and stores API tokens locally so the extension can manage raids on your guild. Sign-in refreshes raids and pools when a guild subdomain is set. Roster download is optional (see Load roster under raid log upload).

    Backup & Restore
    
            
    
                Export all settings to a file — reminders, RaidTick, OpenDKP API (guild, username, roster/pools cache, tick definitions, auction defaults), item watchlist, rank bid limits (from Bidding Tool Bid Rules), EQ log rules/events, and custom sounds. EQ log file handles cannot travel in JSON (browser security)—re-select the log in Loot Monitor after restore on a new profile. Watchlist and API fields auto-save as you type. Download a backup before reloading or updating the extension. Restore after reinstalling or when testing new versions.
            
    
            
    
                
                
            
    
            
    
                
                
    
                    Password and JWT tokens are stored in plain text in the backup file. Store the file somewhere safe—anyone with the file can sign in as you.
